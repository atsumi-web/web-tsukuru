# 堀田建設 採用 LP：公開・運用計画（技術版）

このドキュメントでは、LP の安定した公開と、将来的な管理のしやすさを目的とした技術構成についてまとめています。

## 1. 公開準備

- **対象ファイル**: `recruit/index.html`（メイン LP）
- **公開環境**: サブドメイン（例: `recruit.hottacon.co.jp`）
- **ドキュメントルート**: サーバー側で `recruit` フォルダを指定して公開することを推奨。

## 2. パス管理のルール（重要）

すべてのリソースは、以下の通り**相対パス**で管理されています。これにより、ディレクトリ移動や環境変更に強い構造になっています。

- **CSS**: `css/style.css`
- **JS**: `js/main.js`
- **画像**: `images/xxxx.jpg`

## 3. 将来の複数 LP 運用案

職種別の LP などを追加する際は、`recruit` フォルダ直下にサブフォルダを作成する構成を推奨します。

```text
recruit/
├── index.html          (メインLP：今回の対象)
├── docs/               (設計資料：本ドキュメント等)
├── lp-driver/          (将来用：ドライバー募集等)
└── lp-office/          (将来用：事務職募集等)
```

## 4. 検証項目

- [ ] サブドメインアクセス時に正しく `index.html` が表示されるか。
- [ ] 画像やフォントがリンク切れ（表示されない）になっていないか。
- [ ] フォーム等のリンク先が正しいか。
