# 【マスタープラン】堀田建設 採用 LP 制作・構成設計図

このドキュメントは、これまでのやり取りで決定した戦略、構成、ヒアリング案をすべて集約した「LP 制作の設計図」です。今後の制作フェーズにおいて、一貫性を持った実装を行うためのマスターガイドとなります。

---

## 1. 全体コンセプト

**「私たちが、続けている理由。」**
求職者が抱く「自分にできるか？」という不安に対し、社員の「等身大の葛藤と克服」を提示することで、強い信頼と共感を生む LP を目指します。

## 2. ディレクトリ・技術構成

- **ドキュメントルート**: `recruit/` フォルダ（サブドメイン公開用）
- **メイン LP**: `recruit/index.html`（パスはすべて相対パスに統一）
- **将来の構成**: `recruit/lp-xxxx/` 形式で、職種別 LP をフォルダごとに独立させて追加。

## 3. コンテンツ戦略（3 名の役割）

ヒアリングから得られる素材を、以下の 3 つの不安解消に割り当てます。

| 担当社員   | 役割・ターゲット       | メインテーマ（不安の解消）                     |
| :--------- | :--------------------- | :--------------------------------------------- |
| **菅原様** | **異業種からの転職者** | 「未経験でも、この教育体制ならプロになれる」   |
| **岡安様** | **地元志向・経験者**   | 「地元で働く誇り。大手とは違うやりがいと安定」 |
| **渥美様** | **新人・高卒・若手**   | 「未完成でいい。失敗を許容し育てる文化の証明」 |

## 4. LP 構成案（ページフロー）

1.  **ヒーロービュー**: 3 人の社員の表情と、心に響くメインコピー。
2.  **イントロダクション**: 「良い話ではありません。リアルな声です」という宣言。
3.  **社員インタビューセクション**: 各自の「無理だと思った瞬間」と「克服の言葉」。
4.  **環境・データ**: 残業時間、有給、資格支援など、言葉を裏付ける事実。
5.  **募集要項**: 明確な選考フローと誠実な待遇表示。
6.  **フッター CTA**: 「迷ったら、まずはカジュアルに話しましょう」という低ハードルの誘導。

## 5. 制作時のチェックポイント

- **「具体的セリフ」の使用**: ヒアリングで出た「先輩の温かい一言」を見出しにする。
- **写真のリアリティ**: 笑顔だけでなく、真剣な表情やチームで話している風景を優先。
- **人間力（コンセプト）の浸透**: 施工の技術だけでなく、その裏にある「人の想い」をテキストに込める。
