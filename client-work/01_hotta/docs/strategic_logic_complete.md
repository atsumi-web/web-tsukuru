# 採用 LP 制作の戦略的ロジックと技術的優位性 完全マニュアル

このドキュメントは、クライアントへの提案や、Antigravity へのインプットに使用できる、採用 LP 制作の核心的な戦略ロジックをまとめたものです。

---

## 1. なぜ「立派な公式 HP」があっても「専用 LP」が必要なのか

### 1-1. HP と LP の役割の違い

| 項目           | 公式 HP                                      | 採用 LP                  |
| -------------- | -------------------------------------------- | ------------------------ |
| **目的**       | 企業の信頼性を担保する                       | 応募への意欲を最大化する |
| **対象**       | 取引先、金融機関、顧客など全ステークホルダー | 求職者のみ               |
| **情報の性質** | 網羅的(情報の広さ)                           | 特化的(情報の深さ)       |
| **比喩**       | 百科事典                                     | ラブレター               |

### 1-2. 情報の翻訳

HP の「硬い事実」を、LP で「柔らかい言葉」へ翻訳する。

**HP の表現例:**

- 「創業 70 年」
- 「官公庁工事実績多数」

**LP の表現例:**

- 「道なきところに、道をつくる」というビジョンが、日々の現場でどう体現されているのか
- どんな喜びがあるのかという物語(ストーリー)

### 1-3. 意思決定の最短化

- 余計なリンクを排除した一本道の構成
- ページ上部から下部まで一気に読み進める
- 熱量を下げずに「応募ボタン」まで導く

## 2. 独自の技術構成(コード記述 × クラウド直結)のメリット

### 2-1. 維持費(固定費)ゼロの実現

**従来型:**

- サーバーという「場所」を借りる
- 月々数千円〜数万円の固定費

**独自構成:**

- 最新のクラウド技術を直接活用
- 場所代としての固定費を徹底排除
- **維持費 0 円**

### 2-2. 圧倒的な表示スピード

**従来型(WordPress 等):**

- ページを開くたびにプログラムが動く
- 表示に時間がかかる
- 3 秒以上で離脱リスク

**独自構成:**

- 最適化されたデータを直接届ける
- 「表示 0 秒」を目指す
- スマホ利用者の離脱を防ぐ

### 2-3. 最高峰のセキュリティ

**従来型(WordPress 等):**

- システムを常に動かし続ける
- 攻撃の隙がある
- プラグインやアップデートが必要

**独自構成:**

- システムを動かさず「完成したデータ」だけを配信
- 改ざんや乗っ取りの隙を与えない
- 銀行や世界トップレベルの IT 企業と同じ手法

### 2-4. 資産としての長期価値

**従来型:**

- 数年でシステムが古くなる
- サービス終了で消える可能性

**独自構成:**

- 10 年後も「腐らない」コード
- 企業の純然たる「知的財産」として手元に残る
- いつでも復元・進化が可能

## 3. なぜ WordPress を使わないのか(3 つの戦略的理由)

### 理由 1: セキュリティリスクという爆弾を抱えないため

> 「WordPress は世界で最も普及しているため、常にサイバー攻撃の標的になっています。プラグインや本体のアップデートを怠ると、サイトの改ざんや個人情報漏洩のリスクが非常に高いのが実情です。
>
> 私が提供する構成(Git × 静的サイト生成)は、**『サーバー側に攻撃の隙がない』**非常に堅牢な仕組みです。採用 LP で応募者の個人情報を扱う以上、セキュリティに妥協しないこの構成こそが、企業のリスク管理として正解だと考えています。」

### 理由 2: 表示速度こそが採用の成約率(CVR)を左右するため

> 「WordPress はページを開くたびに裏側でプログラムが動くため、どうしても表示が重くなりがちです。特にスマホで見る求職者は、3 秒以上表示に時間がかかると離脱すると言われています。
>
> 私の書くコードは、余計な処理を一切排除した『表示スピード特化型』です。『開いた瞬間に表示される』。このストレスのなさが、応募への離脱を防ぐ最大の武器になります。」

### 理由 3: 見えない管理費をゼロにするため

> 「WordPress は、サイトを維持するために常にアップデート作業(メンテナンス)が必要で、それを放置するとサイトが壊れたりウイルスに感染したりします。業者はその『見えない手間』のために月額管理費を請求します。
>
> 私の手法は、一度公開すればインフラ自体が朽ちることがないため、何もしなくても安全に存在し続けます。無意味な『保守費用』を削減し、浮いた予算を『採用広告』や『コンテンツの質』に回していただくため、あえて WordPress を選びません。」

## 4. 「サーバー代無料」と「修正費」の矛盾を解消するロジック

### 4-1. 「家賃」と「コンサル料」の違いで説明する

> 「従来の制作会社は、たとえ何もしなくても**『サーバー代・管理費』という名の『家賃』を毎月徴収します**。これは、サイトをネット上に置いておくだけの費用で、成果には直結しません。
>
> 一方、私のスタイルは**『家賃(インフラ代)』は永久にゼロ**です。これは Git や最新のクラウド技術を駆使した私の技術的サービスの恩恵です。
>
> その代わり、もし御社が『もっと応募を増やしたい』『内容をブラッシュアップしたい』と望まれるなら、そこには私の**『知能(戦略立案と修正作業)』への対価**が発生します。
>
> つまり、**何もしなければ維持費は本当にタダ。成果を伸ばしたい時だけ、スポットまたは伴走型で費用をいただく。**これが最も誠実な形だと考えています。」

### 4-2. ランニングコストの概念を再定義する

**一般的な LP 運用:**

- サーバー代(固定費) + 修正費(変動費)

**私の場合:**

- サーバー代(固定費): **0 円**
- 修正・運用代(変動費): **必要な時だけ、または伴走が必要な時だけ**

**メリット:**

- 採用がうまくいっている時期や、予算を抑えたい時期には完全にコストをゼロにできる
- この「止められる自由」があることこそ、クライアント様にとっての真のメリット

### 4-3. コードで書く資産性の真意を伝える

> 「なぜ私が、誰でも触れるノーコードツールではなく、コードで書くのか。それは**『表示速度』と『SEO(検索強さ)』、そして『長期間腐らない耐久性』**を担保するためです。
>
> 無料ツールで作ったサイトは、そのサービスが終了すれば消えますが、コードで書いた LP は、御社の純然たる『知的財産』として手元に残ります。
>
> 『誰でも触れる(=壊れやすい・遅い)』よりも、『プロが厳密に管理する(=強固で速い資産)』。どちらが 100 年続く企業にふさわしいか、という提案です。」

## 5. 2 つのプラン提示で納得感を高める

### プラン A: 完全自律プラン

- **納品後の維持費**: 0 円
- **サーバー代**: 0 円
- **修正**: 必要な時だけ、その都度見積もり(スポット対応)

### プラン B: 採用加速プラン(月額)

- **サーバー代**: 0 円
- **サービス内容**: 月に 1 回のデータ分析と、それに基づくキャッチコピー・写真の入れ替え
- **目的**: 採用単価を下げるための伴走型

**ポイント:**

- どちらを選んでも「サーバー代無料」という看板に嘘をつかない
- クライアントが「自分で選んだ」という納得感を持つ

## 7. クライアントへの FAQ

### Q1. 「HP 改修で十分では?」

**A:**

> 出口を一つに絞る LP とは、成約率において根本的に構造が異なります。HP は「信頼の担保」、LP は「動機形成」という明確な役割分担があります。

### Q2. 「自分で更新したい」

**A:**

> 採用 LP は精密な営業武器です。プロがデータに基づき微調整することで、最大の投資対効果を生みます。
>
> 「自分たちで触れますよ」という耳障りの良い言葉で逃げるのではなく、「プロが責任を持って成果を維持し続ける」という姿勢を大切にしています。

### Q3. 「維持費無料と修正費の矛盾は?」

**A:**

> 「場所代(家賃)」は無料。プロの「知恵と作業(運用)」は有料という明確な分離です。
>
> インフラに払う無駄な固定費を、成果を出すための「戦略」に投資しませんか?
