1. アカウント発行の鉄則ルール

事業がスケールしても困らないよう、「お客様に渡す資産」と「自分が管理するインフラ」を明確に分けます。
迷わず「爆速」でセットアップするためのチェックリストです。

① クライアントごとに「新規作成・取得」するもの

※ 将来的に完全に譲渡可能な「クライアントの資産」です。

  • [ ] Googleアカウント
    作成例： client-name.recruitment@gmail.com
    用途：Googleフォーム（応募管理）、Search Console（解析）、YouTube（動画埋め込み）。
  • [ ] 独自ドメイン
    取得先： お名前.com 等
    「資産価値」の核心です。必ずクライアント名義で取得します。
  • [ ] LINE公式アカウント
    クライアント名義で作成し、渥美さんを「運用担当者」として招待してもらいます。

② 渥美さんのアカウントで「追加・管理」するもの

※ 「編集長」としての作業効率を最大化するインフラです。

  • [ ] GitHubリポジトリ（Private）
    作成場所： 渥美さんのアカウント内
    命名規則： project-clientname-lp
  • [ ] Cloudflare Pagesプロジェクト
    作成場所： 渥美さんのアカウント内
    独自ドメインをここに紐付け、SSL（https）化を即座に完了させます。
  • [ ] Googleアナリティクス（GA4）
    作成場所： 渥美さんの管理アカウント内に「プロパティ」追加
    解析・改善まで丸投げしてもらうなら、一括管理が早いです。
💡 「爆速」のコツ：テンプレート活用

GitHub上に「ベースとなるAntigravityのテンプレート」を1つ作っておき、新しい案件が来たらそれをコピー（Use this template）するだけにすると、構築がさらに速くなります。

3. 【コスト管理】有料プランへの切り替え目安

基本は「無料（Free）」で十分戦えますが、事業規模が大きくなった時の判断基準です。

🐙

GitHub

  • 現状：Freeプラン（0円）
    非公開リポジトリも無制限で作れます。個人〜小規模チームなら最強です。
  • 切り替え時：Teamプラン（約$4/月）
    ✅ 「誤爆防止ロック」をかけたい時
    （例：「間違えて全部消しちゃった！」という事故をシステム的に防ぐ機能。初心者のうちはロックがない方が自由に直せますが、チームでやるなら必須です）
☁️

Cloudflare

  • 現状：Freeプラン（0円）
    帯域無制限、サイト数無制限。神ツールです。100サイトくらいまで余裕でいけます。
  • 切り替え時：Proプラン（約$20/月）
    ✅ 「更新の渋滞」が起きた時
    （無料版は「1つずつ」しか更新できません。えりさんお一人の場合は、同時に2つのPCで別々の案件を更新ボタン押す…なんてことはないので、当分0円で大丈夫です！）
