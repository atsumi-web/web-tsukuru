# GitHub Pages デプロイ手順書（コマンド不要版）

作成日：2026-01-16

## はじめに

VS Code を使って、修正した Web サイトをインターネット上に公開（更新）する手順です。
コマンド（黒い画面）を使わず、ボタン操作だけで完結する方法をまとめました。

---

## 1. 変更を保存する

1.  VS Code でファイルを編集し、保存（`Ctrl + S`）。
2.  左側のメニューから「ソース管理アイコン（枝分かれのマーク）」をクリック。
    - _（ここにアイコンのキャプチャを貼る）_

## 2. 変更をステージング・コミットする

1.  「変更」欄にあるファイル一覧の横の「＋」ボタン（変更をステージング）を押す。
2.  上の入力欄（メッセージ欄）に、変更内容を一言で入力する。
    - 例：「日報のリンクを追加」「画像の差し替え」など
3.  **「コミット」ボタン（青いボタン）をクリック**。

## 3. GitHub へ反映（プッシュ）する

1.  コミット後、同じ場所に表示される**「変更の同期（Sync Changes）」**ボタンをクリック。
2.  「OK」などの確認画面が出たら「OK」を押す。

→ これで GitHub 上のデータが更新されます。

## 4. 公開確認

1.  数分待つ（1〜3 分程度）。
2.  自分の Web サイトの URL にアクセスして、表示が変わっているか確認。
    - 変わっていない場合は、ブラウザの更新ボタンを連打（`Ctrl + F5` でスーパーリロード）してください。
