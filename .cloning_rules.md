# 【サイト模写用】最強のコーディング・ルール

## 1. 模写の基本原則（最優先）

- **「ピクセルパーフェクト」の追求**: デザインの再現を最優先とし、マージン、パディング、フォントサイズ、色、透明度などの数値は元サイトの検証ツール（DevTools）の値を正確に反映すること。
- **勝手なアレンジの禁止**: 元サイトにない要素やスタイルを追加しない。

## 2. HTML構造

- **セマンティックなマークアップ**: `header`, `main`, `section`, `footer`, `nav` などの適切なタグを使用し、文書構造を正しく保つ。
- **クラス命名（BEM）**: クラス名はBEM（Block Element Modifier）に従う。
  - 例: `c-card`, `c-card__title`, `c-card__img--small`

## 3. CSS/スタイル

- **リセットCSSの適用**: 指定のリセットCSSをベースにする。
- **CSS変数の活用**: 「メインカラー」「フォントファミリー」「共通の余白」などを最初に `--primary-color` 等として定義。
- **マジックナンバーの回避**: 元サイトの意図を組み取り、可能な限り一貫性のある数値で管理。

## 4. レスポンシブ対応（マルチデバイス徹底対応）

- **ブレイクポイントの厳守**: 元サイトの切り替わるタイミングを正確に特定。
- **「魔の中間サイズ」の死守**: PC（1200px）からタブレット（768px）に切り替わる際の中間サイズで文字がはみ出したり、レイアウトが崩れたりしないよう、可変ユニット（vw, %）や適切なメディアクエリ（1024px等）を駆使すること。
- **検証の徹底**: 特定のデバイスサイズ（iPhone/Pixel等）だけでなく、ブラウザの幅をシームレスに変更しながら表示崩れがないか常に確認。

## 5. CSS分割管理ルール

- **1セクション1ファイル**: 各セクションごとに専用のCSSファイル（例: `css/message.css`）を作成。
- **メインファイルでの集約**: `style.css` を親とし、そこですべてのセクション用CSSをインポート。

## 6. IIJ Engineering 模写特化ポイント

- **「斜めのデザイン（クリップパス）」**: `clip-path` や疑似要素を使用し、角度を正確に再現。
- **「タイポグラフィと行間」**: `letter-spacing` と `line-height` の再現にこだわる。
- **「ホバーアクション」**: transitionの時間とタイミングを正確に模写。
