# マニュアルテンプレート使用ガイド

## 📁 テンプレート一覧

このフォルダには、新規マニュアル作成時に使用する**UTF-8保証済み**テンプレートがあります。

### 利用可能なテンプレート

1. **`internal_template.html`** - Internal用マニュアル（社内向け技術文書）
2. **`admin_template.html`** - Admin用マニュアル（管理画面・設定系）
3. **`client_template.html`** - Client用マニュアル（クライアント向け）
4. **`daily_template.html`** - Daily用ルーチン（日次作業手順）

---

## ✅ 使い方（3ステップ）

### Step 1: テンプレートをコピー

お客様に合ったテンプレートを選び、目的のフォルダにコピーします。

```powershell
# 例: Internal用マニュアルを作成
Copy-Item "manuals/_templates/internal_template.html" "manuals/internal/新しいマニュアル.html"
```

### Step 2: 内容を編集

コピーしたファイルを開いて、以下の箇所を編集します：

- **【タイトルをここに記入】** → ページタイトルに変更
- **【ページタイトル】** → h1見出しに変更
- **【サブタイトル・説明文】** → 説明文に変更
- 本文を記入

### Step 3: エンコーディング確認

- VSCode右下が「UTF-8」になっているか確認
- 日本語を1文字入力して保存
- ブラウザで開いて文字化けがないか確認

---

## 🛡️ 文字化け防止のポイント

1. **必ずテンプレートからコピー**（ゼロから作らない）
2. **VSCode右下を確認**（UTF-8表示）
3. **ブラウザで確認**（文字化けテスト）
4. **Gitでコミット**（`git add` → `git commit`）

---

## 📝 各テンプレートの特徴

### Internal（社内向け）

- ナビゲーション: `../admin/Private_index.html`
- 用途: 技術文書、コーディング規則、ワークフローなど

### Admin（管理画面）

- ナビゲーション: `admin.html`
- 用途: 管理画面マニュアル、設定手順など

### Client（クライアント向け）

- ナビゲーション: `index.html`
- 用途: クライアント提供マニュアル、使い方ガイドなど

### Daily（日次ルーチン）

- デザイン: カラフルで視覚的
- 用途: 朝・昼・夜のルーチン、繰り返し作業の手順書

---

## 💡 Tips

- テンプレートは**UTF-8 BOMなし**で保存されています
- グローバルナビゲーション（右下の浮遊ボタン）も含まれています
- FontAwesome、Noto Sans JPが自動で読み込まれます

文字化けの心配なく、安心してマニュアル作成を始められます！
