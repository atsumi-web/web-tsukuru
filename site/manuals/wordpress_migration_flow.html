🚀 静的HTMLからWordPressへの完全移行＆仮公開マニュアル

本資料は、完成した静的なHTMLサイトをオリジナルのWordPressテーマに変換し、クライアント確認用のデモ環境へデプロイするまでの、「Webとしくみをつくる」最速構築フローです。

STEP1：テーマ構造の解体とPHP化

完成した静的HTMLをWordPress専用のPHPファイル群に解体・分配します。

必須ファイル構成

Localで作成したテーマフォルダ（例: youchien-theme）内に以下を用意します。

  • style.css：テーマ情報（Theme Nameなど）を記載する宣誓書。
  • functions.php：タイトルタグの自動出力やアイキャッチ画像を有効化する設定書。
  • index.php：専用ページが無い場合に表示される予備の画面。
  • header.php：トップページ上部（ナビゲーションまで）を共通化したもの。
  • footer.php：トップページ下部（フッター以降）を共通化したもの。
  • front-page.php：トップページのメインコンテンツ。
⚠️ パスの書き換えルール（絶対パスの徹底）

WordPressは階層が複雑なため、画像やCSSのパスはすべて「絶対パス」に書き換える必要があります。
./css/style.css
↓変更
<?php echo get_template_directory_uri(); ?>/assets/css/style.css

STEP2：お知らせ機能の動的化（WP_Query）

外部JavaScript（microCMS等）での連携を破棄し、WordPress本体の投稿機能と連動させます。

<?php
// お知らせの最新5件を取得する設定
$args = array(
    'post_type'      => 'post',
    'posts_per_page' => 5
);
$news_query = new WP_Query($args);

// 記事が存在する場合のループ処理
if ( $news_query->have_posts() ) :
    while ( $news_query->have_posts() ) : $news_query->the_post();
?>
        <!-- ここに1件ずつのデザインを記述 -->
        <a href="<?php the_permalink(); ?>"><?php the_title(); ?></a>
<?php
    endwhile;
    wp_reset_postdata(); // 必須：データをリセット
else :
?>
        <p>お知らせがありません。</p>
<?php endif; ?>

STEP3：作業環境（LocalWP）の絶対ルールと切り替え

HTML制作の「Live Server」から、プロのWordPress開発環境へのパラダイムシフトです。

  • ① もう「Go Live」は使わない
    .phpファイルはGo Liveでは正常に動きません。必ず緑色の「Local」アプリの『Start site』を押してエンジンをかけ、『Open site』からブラウザを開いてリロードで確認します。
  • ② Antigravity（VSCode）の「引き出し」を切り替える
    作業時は巨大な web-tsukuru フォルダを開くのではなく、「ファイル ＞ フォルダーを開く」から必ず「該当のテーマフォルダ単体のみ」（例: \wp-content\themes\youchien-theme）を指定して開き直します。
    ※ファイルの保存場所を移動させるわけではなく、VSCodeのカメラをズームインさせるイメージです。

STEP4：即日公開（InstaWPでのデモ提出）

ローカル環境のサイトを、クライアント確認用の仮サーバー（48時間限定）にアップロードします。

  1. データのエクスポート：ローカル環境（Local）に『All-in-One WP Migration』をインストールし、エクスポート機能で .wpress ファイルをダウンロードする。
  2. クラウド環境の構築：InstaWP で新規の空のWordPressを数秒で構築する。
  3. データのインポート：InstaWPで作ったサイトの管理画面を開き、『All-in-One WP Migration』から先ほどの .wpress ファイルをインポートする。これだけで完全移管完了。

💡 困ったときのトラブルシューティング事典

Q. 投稿したお知らせがトップページに出てこない！
A. 投稿ステータスが「下書き（Draft）」になっていませんか？必ず「青い公開（Publish）ボタン」を押してください。
A. InstaWPなどのサーバーの場合、上部の黒いバーにある『Purge Cache』を押してキャッシュをクリアしてください。
Q. いつも通りVSCodeで探しても html ファイルが見つからない！
A. WordPressのテーマフォルダの中に .html は一生存在しません。全ては .php に合体変換されています。デザインを直す時は、VSCodeで front-page.php などを直接開いて中のHTMLタグを編集してください。
Q. 記事をクリックしたら、変な簡素な画面になってしまった！
A. お知らせの詳細ページ用コード（single.php）がまだ未作成のため、万能予備ファイルの index.php がむき出しで表示されています。トップページだけ見せるデモ段階では、リンクの遷移先を一時的に href="#!" 等で無効化しておくのが鉄則です。
