## 2. 実装の「黄金の5ステップ」

### STEP 1：精緻な分析（Gemini ＆ えりさん）

- 元サイトの該当セクションを隅々まで観察。
- Geminiに指示し、「HTML構造の設計案」「CSS変数の候補（色・余白）」「ImageFX用の指示書」をアウトプットさせる。

### STEP 2：アセットの準備（ImageFX ＆ えりさん）

- Geminiが作ったプロンプトをImageFXに入力し、素材を出す。
- 生成した画像は `images/` に保存し、`works-1.jpg` のようにAntigravityが認識しやすい名前をつける。
- **重要**: 画像は必ず [Squoosh](https://squoosh.app/) で軽量化してから保存する。

### STEP 3：骨組みの実装（Antigravity）

- **HTML**: BEM（Block Element Modifier）を徹底し、セマンティックなタグで構造を作る。
- **CSS**: セクションごとに `.css` ファイルを独立させ、`style.css` でインポートする。

### STEP 4：スタイルの作り込み（Antigravity ＆ 元サイト）

- `variables.css` の変数を使い、マジックナンバーを排除。
- フォントサイズ、行間、字間を1px単位で合わせる。
- 斜め背景（`clip-path`）やホバーアニメーションを実装。

### STEP 5：魔の中間サイズ攻略（Antigravity ＆ えりさん）

- **1200px〜768px** の「タブレット・ノートPC」表示を徹底検証。
- メニューが改行される前にハンバーガーメニューへ切り替える。
- カードが溢れる前に `grid-template-columns` を調整。
- `visibility: hidden` や `pointer-events: none` でメニューのチラつきを殺す。

---

## 3. 絶対死守すべき「最強の鉄則」

### A. ナビゲーションの堅牢性

- PCメニューが「いっぱいいっぱい」になる前の、余裕のある幅（例: 1200px）でモバイル形式に逃がす。
- JSとしきい値（innerWidth）を完全に同期させ、挙動の矛盾をゼロにする。

### B. BEMによる命名規則

- `block__element--modifier` の形式を崩さない。
- 汎用的な `.card` ではなく、`.work-card` などのセクション特有の命名を優先する。

### C. マジックナンバーの禁止

- `margin: 114px` のような謎の数値は避け、`var(--spacing-xxxl)` など意味のある変数で管理する。

## 4. セクション別・模写のヒント

| セクション型   | 模写のコツ                                                                       |
| :------------- | :------------------------------------------------------------------------------- |
| **MESSAGE型**  | `clip-path` で大胆な斜め背景を作り、大きな背景テキストで空間を埋める。           |
| **TRAINING型** | 特徴的なバッジ（-45度回転）と、`flex-basis` を使った段階的なタイムライン。       |
| **VOICE型**    | 中間サイズでの「アバター＋テキスト」のカード崩れを防ぐため、早めに縦並びへ移行。 |
| **DATA型**     | 数字の強調と、単位の見せ方（baseline揃え）にこだわる。                           |
