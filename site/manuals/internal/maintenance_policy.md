# 保守管理ポリシー

## 基本方針

**Git管理での納品を標準とし、保守契約はGit管理のみ受ける**

---

## 1. 受ける保守契約

### ✅ Git管理（推奨）

**対象プラットフォーム**:

- Cloudflare Pages（メイン）
- GitHub Pages

**メリット**:

- `git push` だけで更新完了
- 自動デプロイ、自動バックアップ
- クライアントが勝手にファイルを触れない
- えりさんの作業時間: 最小限

**保守料金目安**:

- 月額 1〜3万円
- 更新作業込み

---

## 2. 受けない保守契約

### ❌ レンタルサーバー（FTP管理）

**理由**:

- FTPログインの手間
- クライアントが勝手にファイルを触るリスク
- サーバー環境がクライアントごとにバラバラ
- バックアップ管理が面倒
- 時間コストが高すぎる

**対応**:

- 納品時に「保守契約は提供しない」と明示
- 緊急対応のみ、別途見積（高めに設定）

---

## 3. クライアントへの提案フロー

### 納品時の選択肢

#### プランA: Git管理での納品（推奨）

「Webサイトを最新技術で管理し、将来的な保守も安価にご提供できます」

**メリット**:

- 高速・安全・自動更新
- 継続的な保守契約が可能
- ドメイン設定はクライアント側

#### プランB: レンタルサーバー納品

「従来型のレンタルサーバーでの納品も可能ですが、保守は基本的に提供しておりません」

**メリット**:

- クライアントが自社で完全管理できる
- えりさんは納品後、関与しない

---

## 4. 実装例: 堀田建設

### 推奨構成

1. **GitHub** でソースコード管理（えりさんが管理）
2. **Cloudflare Pages** で自動デプロイ
3. クライアントのサブドメイン `recruit.hotta-kensetsu.co.jp` を設定
4. えりさんが `git push` するだけで更新完了

### 保守契約の提案

- 月額 2万円
- 年間契約で 20万円（2ヶ月分無料）
