# CSS管理ガイドライン

## 基本原則：単一の情報源（Single Source of Truth）

**すべてのスタイル定義は`renewal.css`のみに記述する。**

## インラインCSSの役割

`index.html`の`<style>`タグ内には**クリティカルCSSのみ**を記述：

### 許可される内容

- ✅ `@font-face`定義（フォント読み込み）
- ✅ CSS変数（`:root`）
- ✅ 基本的なレイアウト（`html`, `body`）
- ✅ タイポグラフィの基本設定（見出し、段落のフォントファミリー、ウェイト）

### 禁止される内容

- ❌ コンポーネントスタイル（`.section-badge`, `.cta-button`など）
- ❌ ヘッダー、ナビゲーション、フッターのスタイル
- ❌ レスポンシブルール（`@media`クエリ）
- ❌ アニメーション（`@keyframes`）
- ❌ 個別のセクションスタイル

## CSS編集時のルール

1. **コンポーネントやレイアウトの変更は`renewal.css`のみを編集**
2. **編集後は`renewal.min.css`を同期**（コピー）
3. **インラインCSSは絶対に触らない**（フォント定義や変数以外）

## CSS同期コマンド

```powershell
# renewal.cssの変更をminified版に反映
Copy-Item -Force "css\renewal.css" "css\renewal.min.css"
```

## 重複チェック方法

定期的にインラインCSSをレビューし、以下が含まれていないか確認：

```bash
# インラインCSSで禁止パターンを検索
# .section-badge, .container, .cta-buttonなどのクラス定義
```

## この規則の理由

- **保守性向上**：修正箇所が一箇所で済む
- **一貫性確保**：重複による不整合を防ぐ
- **パフォーマンス**：インラインCSSは最小限（初期表示の高速化）
