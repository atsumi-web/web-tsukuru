# /update-rules ワークフロー

このコマンドを実行すると、今日の会話から得た知見を自動的に整理して
`09_coding_rules.html` と `coding_standard.md` に追記します。

## 手順

1. 今日の会話を振り返り、以下のカテゴリで「新しく学んだこと」を抽出する：
   - バグの根本原因と解決策
   - CSSの詳細度・カスケードに関する発見
   - モバイル/デスクトップ表示の注意点
   - デプロイ・ツール操作の知見
   - 「やってはいけないこと」と「その代わりにやること」

2. 抽出した知見を以下の形式で `09_coding_rules.html` の「困った時の事例集」セクションに追記する：
   - **症状**：どんな問題が起きたか（「モバイルで右側が見切れる」など）
   - **原因**：何が問題だったか
   - **解決策**：どう直したか（コードスニペット付き）

3. 同じ内容を要約して `coding_standard.md`（KI）にも追記する。

4. 09_coding_rules.html に「困った時の事例集」セクションがまだない場合は新規に作成する。
   セクションは既存のスタイル（`.rule-box.good` / `.rule-box.bad`）に合わせること。

5. 変更をコミット・プッシュする：
   ```
   git add -A
   git commit -m "docs: update coding rules with lessons learned YYYY-MM-DD"
   git push
   ```

## 注意事項

- 「些細なこと」ほど価値がある。当たり前に見えても、ハマったことは全て記録する。
- 事例は「症状ベース」で書く（原因ベースではなく）。「sticky使った後にmobileでずれる」→ 検索しやすい。
- コードスニペットは最小限に。長すぎると読まれなくなる。
