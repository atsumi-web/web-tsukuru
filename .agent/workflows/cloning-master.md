# 🏆 サイト模写（クローニング）最強マスター要件

このワークフローは、指定されたWebサイトを「ピクセルパーフェクト」かつ「BEM完全準拠」で模写するための絶対ルールである。Gemini（構成分析）、ImageFX（画像生成）、Antigravity（コード構築）のAI連携シナジーを前提とする。

## Ⅰ. 根幹の設計思想（絶対遵守事項）
- **脱マジックナンバー:** `margin: 114px` のような謎の直打ちを禁ずる。すべて `variables.css` のCSS変数（例: `var(--spacing-xxl)` や `var(--color-primary)`）を用いて一元管理すること。
- **BEMによる厳格な命名:** `block__element--modifier` の規則を崩さないこと。汎用的すぎる `.card` ではなく、機能を明記した `.works-card` や `.message-title` としてカプセル化する。
- **セクション別CSS完全分離:** 1セクション1ファイルを徹底する（例: `css/hero.css`, `css/message.css`）。これらのファイルは必ず親となる `style.css` (または `index.css`) で `@import` して読み込む構造とする。

## Ⅱ. 黄金の実装ステップ（5段階）

### Step 1: 精緻な分析と言語化（構造分析）
- 模写元の余白、フォント種類、フォントサイズ、ホバー挙動を1px・0.1秒単位で徹底的に観察する。
- この時点で必要なCSS変数（カラーパレットの抽出、ブレイクポイントの設定）を抽出し宣言する。

### Step 2: アセットの錬成（画像準備）
- 構成に必要な画像や背景素材（ImageFX等で生成されたもの）は必ず `images/` ディレクトリに保存する。
- アセットは事前に軽量化（Squoosh等を使用し `webp` 形式化）されていることを前提としてコーディングする。

### Step 3: 鋼の骨組み（HTML/BEM構築）
- セマンティックなHTML5タグ (`header`, `main`, `section`, `nav`, `footer` など) とBEMに基づくクラス構文で堅牢な骨格を記述する。
- Google Fonts などの外部読み込みが必要な場合は、必ずプリロード（`preconnect` と `preload`）を設定しパフォーマンス低下を防ぐ。

### Step 4: 1pxの彫刻（緻密なスタイリング）
- セクション境界に斜めや波型のデザインがある場合は `clip-path` や疑似要素を駆使して完全再現する。
- タイポグラフィ（`letter-spacing`, `line-height`, `font-weight`）を直感ではなく数値に基づいて元サイトと完全同調させる。
- ホバー時やスクロール時の `transition` のミリ秒数やイージング関数（`cubic-bezier`等）すらも一切妥協せず模写する。

### Step 5: 「魔の中間サイズ」の完全制圧（マルチデバイス対応）
- **1200px〜768px（タブレット・小型ノートPC）** で閲覧した段階での「カードの溢れ」「テキストの不自然な折り返し」は絶対に許されない。
- ヘッダーのナビゲーションメニュー項目が溢れる前に、余裕のあるブレイクポイント（例: 1200px以下）で直ちにハンバーガーメニューへ転換させること。
- メニュー非表示時などの状態制御には `transform` だけでなく `visibility: hidden` や `pointer-events: none` を併用し、透明な層による誤クリックやチラつきのバグを完全に無効化する。
- JavaScriptでの判定しきい値（`window.innerWidth`）とCSSのメディアクエリ（`@media`）のブレイクポイントは完全に数値を同期させ、挙動の矛盾をゼロにする。

## Ⅲ. 品質担保の最終セルフチェックリスト
- [ ] ブラウザの幅をマウスでドラッグして連続的（無段階）にリサイズした際、いずれの幅でもレイアウトが0.1秒たりとも破綻していないか？
- [ ] PC、タブレット、スマホのそれぞれで、フォントサイズや余白が `clamp()` 等で滑らかに最適化されているか？
- [ ] BEMのネスト（入れ子）が深すぎるせいで、後からの修正（保守性）が極端に落ちていないか？
