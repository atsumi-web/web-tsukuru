## コーディングルール

### 1. CSS構造の原則

- **シングルソースオブトゥルース**: すべてのスタイルは原則として `css/renewal.css` に集約する
- **コンポーネント指向**: クラス定義のすぐ直下に、そのクラスに関するメディアクエリ（@media）を記述する
- **!important の禁止**: 原則として `!important` は使用しない。詳細度の問題は、`:where()` セレクタによるリセット、またはHTML構造の見直しで解決する

### 2. デザイントークンの使用

- **変数の徹底活用**: 色、数値（padding, gap）、フォント名はすべて `:root` で定義された変数を使用する
- **ハードコードの禁止**: `#ffffff` や `12px` などの値を直接各セレクタに記述しない。必要に応じて `:root` に新しい変数を定義してから使用する
- **動的パディング**: セクションの上下余白など、画面幅に依存する数値は `clamp()` 関数を使用して滑らかに可変させる

### 3. 命名規則と状態の管理

- **命名の一貫性**: 新しいクラス名を作成する際は、既存の命名規則（`.cta-button`, `.price-card` 等）を逸脱しない
- **ステート（状態）のセット記述**: 要素の色やスタイルを変更する場合、必ず `:hover`, `:active`, `::before/after` などの関連する状態もセットで確認・修正する
- **セルフクリーンアップ**: HTMLから要素が削除された場合、対応する不要になったCSSセレクタも速やかに削除候補として提案または削除する

### 4. コードの物理的管理

- **追記ではなく「上書き」**: 新しいスタイルが必要な場合、既存のクラス定義を探して直接編集する。ファイルの末尾に新しいルールを追記しないこと
- **意味のあるコメント**: セクションの区切りや、複雑なアニメーションの意図を日本語で明記する

### 5. 文字化け防止（最重要）

- **UTF-8 BOM無し**で必ず保存する（Windows環境で特に注意）
- **日本語コメント**を含むファイルは必ずUTF-8であることをツールで確認してから保存
- **PowerShellでの文字化け確認コマンド**:
  ```powershell
  Get-Content -Path "ファイル名" -Encoding UTF8
  ```
- **絶対にMojibakeを起こさない**（文字化けはプロフェッショナルの信頼を損なう）

### 6. パフォーマンス最適化（成約率に直結）

- **画像最適化は必須**:
  - `srcset` と `sizes` 属性を必ず使用
  - `loading="lazy"` で遅延読み込み（ファーストビュー以外）
  - WebP形式を優先、フォールバックでJPG/PNG
- **CSS/JSの最小化**:
  - `renewal.min.css` への同期を忘れない
  - 不要なCSSセレクタは削除
- **1秒の遅延で7%のコンバージョン低下**を常に意識

### 7. モバイルファースト（採用LP向け必須）

- **スマホ閲覧率が高い**ため、モバイル表示を最優先
- **タップ領域は最小44px**（CTAボタン、電話ボタンなど）
- **フォントサイズは16px以上**（モバイルでズーム不要）
- **レスポンシブテスト必須**:
  - 375px（モバイル）
  - 768px（タブレット縦）
  - 1024px（タブレット横）
  - 1200px以上（PC）

### 8. アクセシビリティ（SEOにも影響）

- **alt属性は必須**（空でも `alt=""` を明示）
- **aria属性**:
  - `aria-label` でボタンの目的を明示
  - `aria-expanded` でアコーディオンの状態管理
- **カラーコントラスト**:
  - 本文: 4.5:1以上（WCAG AA）
  - CTAボタン: 3:1以上
- **キーボード操作**:
  - Tab順序が論理的
  - フォーカス状態が視覚的に明確

### 9. CTAの最適化基準

- **ボタンサイズ**: 高さ44px以上（タップしやすさ）
- **色**: アクションを促す目立つ色（緑、オレンジなど）
- **配置**: ファーストビューに1つ、スクロール後に複数配置
- **文言**: 行動を促す動詞（「応募する」「質問する」「今すぐ登録」）

### 10. 作業フロー

デザイン修正の依頼があった際は以下のステップで実行する：

1. **最小限のコード変更で目的を達成する（DRY原則）**
2. **既存の構造を壊さず、メンテナンス性の高い美しいコードで出力する**
3. **修正完了後、コード全体のインデントや整理が保たれているかセルフチェックする**
4. **成約率（応募数）に直結する変更から優先的に実行する**

### 11. インラインCSSの扱い

- **✅ インラインCSS（index.html内の`<style>`）に書いて良いもの**:
  - @font-face 定義
  - CSS変数（:root）
  - 基本レイアウト（html, body）
- **❌ 外部CSS（renewal.css）に書くべきもの**:
  - コンポーネントスタイル（.section-badge, .cta-buttonなど）
  - レスポンシブルール（@media queries）
  - アニメーション（@keyframes）

### 7. CSS同期

- CSS修正後は必ず `renewal.min.css` に同期する：
  ```powershell
  Copy-Item -Force "css\renewal.css" "css\renewal.min.css"
  ```
